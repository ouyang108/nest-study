# 项目协作规则（Claude Code 自动加载）

## README.md 编辑读取策略

README.md 已 1300+ 行，编辑时**不要大段读全文**，避免浪费 token。

### 硬性约束

- 单次 `Read` 控制在 **80 行以内**
- 不把 README 全文存任何地方（行号会漂移，存了也不准）

### 操作清单

| 场景 | 做法 |
| --- | --- |
| 改某一节内容 | 先 `Grep "^#{1,4} "` 抓所有标题定位行号，只 `Read` 目标节附近 |
| 在已知锚点附近插入/替换 | 直接 `Edit` 锚点字符串，不读全文 |
| 整体结构调整（重排、降级标题） | 用 `Grep` + 脚本（awk/head/tail），不读全文 |
| 用户说「优化 X 章节」 | 先 `Grep ^#` 抓所有标题（几十行），再只 `Read` 目标章节 |

### 禁止行为

- ❌ 因为「想确认整体结构」就读全文 → 用 `Grep "^#{1,4} "` 抓标题代替
- ❌ 把 README 全文复制到对话或 memory 里「为下次省事」
- ❌ 单次 `Read` 超过 200 行（80 行是软上限，200 行是硬红线）

类似规则也适用于其他大文件（>500 行）。

---

## 代码注释要求

- 生成的代码**必须加注释**（继承自全局 CLAUDE.md）
- 注释解释 **WHY**，不解释 WHAT（命名应该已经说清楚做什么）
- 非显而易见的设计决策、踩过的坑、特殊约束才值得写注释
